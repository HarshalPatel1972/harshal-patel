## 2025-03-27 - [useTilt Layout Thrashing]
**Learning:** `useTilt` in `AnimationKit.tsx` was performing synchronous DOM reads (`getBoundingClientRect()`) and writes (`style.transform` and `style.transition`) inside high-frequency `mousemove` event listeners without any throttling or requestAnimationFrame. This is a classic anti-pattern that causes layout thrashing and main thread blocking, as the browser is forced to recalculate layout repeatedly on every pixel movement.
**Action:** When creating high-frequency mouse tracking hooks, use `requestAnimationFrame` to batch the read-write operations, cache coordinates in a local variable or ref outside the `requestAnimationFrame` call to ensure the latest position is used, and make sure to clean up the RAF id on mouse leave or unmount.
