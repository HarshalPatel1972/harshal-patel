## 2024-05-19 - [React Canvas Coordinates Anti-pattern]
**Learning:** Passing mouse coordinates from high-frequency `mousemove` events to a child Canvas component using React state (e.g., `useState({x, y})`) causes excessive re-renders of the parent. Additionally, using those coordinates as dependencies in the Canvas `useEffect` loop tears down and restarts the `requestAnimationFrame` loop unnecessarily on every pixel movement.
**Action:** When passing continuous interactive coordinates (like mouse position) to an HTML Canvas component that manages its own animation loop, always pass a mutable `useRef` containing the coordinates rather than React state. This prevents cascading re-renders and preserves the internal animation loop's closure without triggering lifecycle restarts.
