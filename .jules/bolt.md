## 2024-06-25 - [Performance Improvement] Hoisting Static Array Geometry Rendering
**Learning:** For truly static arrays (e.g., SVG grids, purely visual background shapes) that do not depend on props or state, defining them via inline `Array.from()` inside the React component body or even with `useMemo()` can cause redundant execution, allocations, and `react-hooks/purity` lint errors (when using impure functions like `Math.random()`).
**Action:** Always extract and hoist these static array definitions to constants outside the React component body. This provides a measurable performance gain by preventing regeneration on every render cycle, reduces garbage collection pressure, and cleanly resolves linting issues associated with impure calculations inside React's render phase.
