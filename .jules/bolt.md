## 2024-04-10 - Bot Detection and Pipeline Optimization in Edge Route
**Learning:** For high-frequency edge API routes (like `visitor-count`), processing User-Agent strings using `Array.prototype.some` and `.includes()` combined with `.toLowerCase()` introduces measurable CPU overhead per request. Similarly, running multiple sequential Redis commands (`sadd`, `incr`, `scard`, `get`) incurs excessive network latency round-trips. When moving to a pipeline via ioredis, results are returned as a nested array format (`[[error, result]]`) requiring explicit index-based error checking and extraction.
**Action:** Replace iterative string checks with a pre-compiled `RegExp` with the `i` flag hoisted outside the route handler. Use `.pipeline()` for multiple Redis operations and extract correctly formatted tuples `results[index][1]` to safely retrieve batched values while preventing unexpected undefined/null crashes.
