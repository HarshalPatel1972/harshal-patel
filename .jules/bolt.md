## 2024-04-11 - Synchronous State Updates in High-Frequency Interactive Events Create Cascading Render Overhead
**Learning:** In `FloatingGallery`, calling `setState` inside a high-frequency `mousemove` listener without rate limiting (e.g., `requestAnimationFrame`) causes synchronous cascading re-renders across the entire gallery layout on every pixel move. The problem is amplified when the parent component holds this state, as it forces all nested children (such as heavily styled or animated `FloatingCard` elements) to re-render.
**Action:** Always batch top-level React state updates resulting from high-frequency DOM events (like scroll or mousemove) using `requestAnimationFrame`. Use a `ticking` ref to drop intermediate events and a `latestEvent` ref to store coordinates mutably rather than letting stale values accumulate in the event closure.
