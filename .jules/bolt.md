## 2025-02-27 - Synchronous Layout Reads and State Updates in High-Frequency Events
**Learning:** Found severe layout thrashing in `useMagnetic`, `useTilt`, and `useParallax` hooks (`src/components/AnimationKit.tsx`) because they perform synchronous layout property calculations (`getBoundingClientRect`) on every `mousemove` or `scroll` event. Additionally, `Hero.tsx` calls `setState` synchronously on `scroll` events leading to cascading React renders.
**Action:** Always batch DOM reads (`getBoundingClientRect`) and React state updates (`setState`) inside `requestAnimationFrame` using a `ticking` boolean flag when handling high-frequency events to prevent layout thrashing and main thread blocking. When batching mouse coordinates, use mutable variables to pass the latest `clientX` and `clientY` to the `rAF` callback to prevent desync.
