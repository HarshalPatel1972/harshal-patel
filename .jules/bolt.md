## 2024-05-24 - Static Array Hoisting Performance Pattern
**Learning:** For static arrays generated purely for visual effects (e.g., SVG grid lines, animated background shards), using `useMemo` or inline `Array.from()` inside a React component adds unnecessary closure overhead and memory allocation during the render cycle. Furthermore, if the generation involves impure functions like `Math.random()`, it can cause `react-hooks/exhaustive-deps` or purity lint warnings if not carefully managed.
**Action:** Always hoist purely static array generations (that don't depend on props or state) outside the component definition as top-level constants. This avoids recalculation on re-renders, reduces garbage collection pressure, and cleanly separates static view data from component logic.
