## 2024-05-10 - [Hoisting Static Arrays to Avoid Purity and Re-render Issues]
**Learning:** For React components rendering complex graphical layouts (e.g. SVG grids, data lines, or shards) using `Array.from` inside `useMemo` with impure functions (like `Math.random()`), ESLint triggers `react-hooks/purity` errors. Additionally, `useMemo` is not fully guaranteed to avoid re-execution in React. By moving perfectly static arrays (even those that contain random generations at initialization) completely outside of the functional component into file-scoped constants, we can completely eliminate garbage collection pressure during renders, improve initial render performance, and definitively solve the purity lint warnings.
**Action:** When a static graphical array (like SVG lines, random shards, or nodes) does not depend on component props or state, always define it as a constant outside the component block instead of relying on `useMemo`.
