## 2024-05-24 - Pre-compiled RegExp and ioredis pipeline batching
**Learning:** In NextJS API routes interacting with Redis, performing sequential `await kv.method()` calls introduces unnecessary network round-trips latency, which significantly impacts performance. Additionally, validating request headers like `user-agent` against a static list of keywords using `Array.prototype.some` and `String.prototype.includes` consumes extra CPU cycles and requires redundant `.toLowerCase()` string conversions.
**Action:** Next time, always check if multiple sequential Redis operations can be batched using `kv.pipeline().exec()` to collapse them into a single network round-trip. For high-frequency string matching against static keyword lists, use a pre-compiled `RegExp` with the `i` (case-insensitive) flag instantiated outside the request handler.
