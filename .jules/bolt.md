## 2026-04-26 - Unthrottled Scroll Listeners Cause Thrashing\n**Learning:** High-frequency event listeners (like scroll or mousemove) in React that trigger direct DOM manipulations or CSS custom property updates without `requestAnimationFrame` throttling can cause layout thrashing and drop main thread FPS significantly. I noticed this anti-pattern in `src/components/Hero.tsx` where the scroll event isn't wrapped in `requestAnimationFrame`.\n**Action:** Always wrap style updates in high-frequency event handlers inside a `requestAnimationFrame` call and protect it with a `ticking` boolean flag to avoid queuing up redundant frames.
