## 2024-05-24 - [Pre-Compiled RegEx for API Route Performance]
**Learning:** Found a common anti-pattern where an array of static bot keywords (`['bot', 'spider', ...]`) was being evaluated iteratively per-request using `Array.prototype.some` and `String.prototype.includes`, along with a redundant `.toLowerCase()` transformation on the incoming `user-agent` string. This approach incurs overhead from allocating new strings and executing iterative matching loops for every hit to the visitor count API route.
**Action:** Replaced the iterative array string checking with a single, pre-compiled native `RegExp` (with the `/i` case-insensitive flag) defined outside the request handler context. This provides optimized, single-pass matching that executes more efficiently at the V8 engine level, reducing CPU cycles spent verifying if incoming requests are bots. Always use pre-compiled regex outside of request loops for static list lookups.
