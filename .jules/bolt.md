## 2024-04-12 - Redis Pipeline Return Format Variability
**Learning:** When using `@vercel/kv` (or `@upstash/redis`), the `.pipeline().exec()` method returns a flat array of values (e.g., `[1, 1, 42, 100]`), unlike the standard `ioredis` behavior which returns an array of tuples `[error, result]` (e.g., `[[null, 1], [null, 42]]`). Blindly indexing `results[2]?.[1]` assuming the `ioredis` format in a Next.js environment will evaluate to `undefined` and cause silent logic failures (e.g., returning 0 hits).
**Action:** When extracting pipeline results in an environment where the Redis client abstraction is not strictly guaranteed to be `ioredis`, always use a normalization helper like `const extractResult = (res: any) => Array.isArray(res) ? res[1] : res;` to safely handle both flat and tuple-based return formats.
