## 2024-05-03 - Memoize Expensive String Manipulations in High-Frequency React Renders
**Learning:** Performing array manipulations like `.split()` and `.map()` inline inside a React component's return statement forces these operations to execute on *every single render cycle*. In components that re-render frequently (e.g., those driven by high-frequency scroll listeners or animations, like `Hero.tsx`), this leads to redundant object and array allocations, increasing CPU cycles and garbage collection pressure, which can cause micro-stutters.
**Action:** Extract expensive data transformations out of the JSX and wrap them in a `useMemo` hook with the appropriate dependency array. This ensures the computations are performed only when the underlying data changes, rather than on every render cycle.
