## 2024-05-18 - Prevent synchronous layout thrashing in Hero scroll engine\n**Learning:** High frequency events like `scroll` can cause significant UI freezes (layout thrashing) when reading layout properties (like `offsetTop`, `offsetHeight`) and directly mutating styles synchronously.\n**Action:** Use a `.jules/bolt.md` to note layout thrashing preventions: pre-calculate static DOM layout measurements inside `useEffect` (updating them on window `resize`), and throttle actual style assignments using `requestAnimationFrame` with a `ticking` boolean.
